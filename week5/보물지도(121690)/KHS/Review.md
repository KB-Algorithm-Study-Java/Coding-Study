3. 함정을 만날 시 한 번 점프가 가능함, 2칸을 뛰어 넘어도 이동 시간은 1로 처리
4. 탈출지점에 도착했을 때 최단 시간은?

**문제해결**
```
접근 1.
마법을 쓴 상태와 안 쓴 상태의 관리가 중요함
[상태 관리]
1. 마법을 안 쓴 경우(해당 칸이 0이며, 방문을 안 한 상태를 가지고 있어야 함
	 a. 마법을 안 쓴 경우 큐 안에 객체는 마법을 안 쓴 상태인 1을 가지고 있음
			- 언제든 함정을 만나면 뛰어 넘을 준비를 함
2. 마법을 써야하거나 쓴 경우(해당 칸이 1이며, 방문을 안 한 상태와, 객체가 마법을 안 쓴 상태)
	 b. 위 조건을 만족 못하면 함정칸을 쳐다 보지도 못함
	 c. 만약 조건을 만족 시 객체를 마법에 쓴 상태로 바꿔주고, 함정 칸을 방문 상태로 바꿔줌
	 
변수 선언(이 부분에 대한 힌트만 받음 boolean을 3차원으로 다루고, 큐 객체에 마법 사용 상태를 관리하라고)
방문 상태 - boolean[n][m][2] -> 이렇게 해서 방문 상태와 마법 상태를 다루면 될듯
객체 상태 - new int[] {좌표,좌표, 거리, 마법 사용 여부} -> 이렇게 관리해서 1인 칸을 만나도 마법을 사용했으면 들어가지 못하게 관리

접근 2.
접근 1의 방식은 거의 맞음 하지만 방문 배열의 용도를 잘 못 사용함
땅따먹기 처럼 이제 생각을 했음 이렇게 생각했다면 방문 배열은 애초에 2차원이여도 충분함
여기서 발생한 문제는 탈출구가 홀에 막혔을 시 마법을 사용할 수 있는 객체가
마법을 사용해 그 앞에 도착을 먼저 한 객체의 방문도장으로 인해 접근을 못하는 경우 가 발생
방문 배열의 관리를 마법을 쓴 객체의 길과 안 쓴 객체의 길로 나눠 생각하는 방식으로 다시 접근
```
