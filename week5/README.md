# 🌲 DFS (Depth First Search)

DFS는 **한 방향으로 끝까지 깊게 탐색한 뒤, 더 이상 갈 곳이 없으면 되돌아오는 탐색 방식**이다.

- Depth First Search
- 깊이를 우선적으로 탐색
- 보통 재귀 또는 Stack 사용
- 경로 탐색, 백트래킹 문제에 자주 사용

👉 핵심: **"일단 끝까지 가본다."**

---

## 🛠️ DFS 핵심 요소

| 개념 | 설명 |
|------|------|
| 방문 처리 | 이미 방문한 노드 재탐색 방지 |
| 재귀 호출 | 다음 노드를 깊게 탐색 |
| 백트래킹 | 탐색 후 이전 상태로 복귀 |
| 종료 조건 | 무한 재귀 방지 |

---

## 💡 DFS 사용 예시

- 그래프 탐색
- 백트래킹
- 경로 찾기
- 사이클 탐지
- 완전 탐색

---

## 📌 DFS 흐름

1. 현재 노드 방문
2. 방문 가능한 다음 노드 탐색
3. 끝까지 이동
4. 더 이상 갈 곳이 없으면 이전 노드로 복귀

---

# 🌊 BFS (Breadth First Search)

BFS는 **현재 위치에서 가까운 곳부터 차례대로 탐색하는 방식**이다.

- Breadth First Search
- 너비 우선 탐색
- Queue 사용
- 최단 거리 문제에서 자주 사용

👉 핵심: **"가까운 곳부터 차례대로 탐색한다."**

---

## 🛠️ BFS 핵심 요소

| 개념 | 설명 |
|------|------|
| Queue | 먼저 들어온 노드부터 탐색 |
| 방문 처리 | 중복 방문 방지 |
| 레벨 탐색 | 같은 거리 단위로 탐색 |
| 최단 거리 | 가중치 없는 그래프에서 최단 거리 보장 |

---

## 💡 BFS 사용 예시

- 최단 거리 탐색
- 미로 문제
- 네트워크 연결
- 레벨 단위 탐색
- 전염 / 확산 시뮬레이션

---

## 📌 BFS 흐름

1. 시작 노드를 Queue에 삽입
2. Queue에서 하나 꺼내기
3. 연결된 노드 탐색
4. 방문하지 않은 노드를 Queue에 삽입
5. Queue가 빌 때까지 반복

---

# 🔄 BFS vs DFS

| 탐색 방식 | 특징 |
|-----------|------|
| DFS | 깊게 탐색 후 되돌아감 |
| BFS | 가까운 곳부터 탐색 |
| DFS 사용 자료구조 | Stack / 재귀 |
| BFS 사용 자료구조 | Queue |
| DFS 특징 | 경로 탐색, 완전 탐색 |
| BFS 특징 | 최단 거리 탐색에 유리 |

---

# ⚠️ 탐색 문제에서 중요하게 볼 것

| 체크 포인트 | 설명 |
|-------------|------|
| 방문 처리 | 중복 탐색 방지 |
| 그래프 표현 | 인접 리스트 / 인접 행렬 |
| 탐색 순서 | BFS인지 DFS인지 판단 |
| 종료 조건 | 무한 루프 방지 |
| 시간복잡도 | O(V + E) 이해하기 |

---

# 💡 정리

| 개념 | 핵심 특징 |
|------|-----------|
| DFS | 한 방향으로 끝까지 탐색 |
| BFS | 가까운 노드부터 탐색 |
| Stack | DFS 구현에 사용 |
| Queue | BFS 구현에 사용 |
| 방문 배열 | 중복 탐색 방지 |
| 그래프 탐색 | 노드와 간선 관계 탐색 |
