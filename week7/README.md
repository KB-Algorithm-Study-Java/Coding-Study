# 📚 DP (Dynamic Programming)

DP는 **큰 문제를 작은 문제로 나누어 해결한 결과를 저장하여 중복 계산을 줄이는 알고리즘 기법**이다.

이미 계산한 값을 재사용하여 시간복잡도를 크게 줄일 수 있다.

* Dynamic Programming
* 동적 계획법
* 중복되는 부분 문제 활용
* 계산 결과 저장
* 최적 부분 구조 이용

👉 핵심: **"한 번 계산한 결과는 다시 계산하지 않는다."**

---

## 🛠️ DP 핵심 요소

| 개념       | 설명                  |
| -------- | ------------------- |
| 중복 부분 문제 | 같은 문제가 반복적으로 등장     |
| 최적 부분 구조 | 작은 문제의 최적해로 큰 문제 해결 |
| 메모이제이션   | 계산 결과 저장            |
| 점화식      | 문제를 작은 문제로 표현       |

---

## 📌 DP 적용 조건

### 1️⃣ 중복 부분 문제 (Overlapping Subproblem)

같은 계산이 여러 번 반복되는 경우

예시)

```text
f(5)
├─ f(4)
│  ├─ f(3)
│  └─ f(2)
└─ f(3)
```

`f(3)`이 여러 번 호출된다.

---

### 2️⃣ 최적 부분 구조 (Optimal Substructure)

큰 문제의 최적해를 작은 문제의 최적해로 구성할 수 있는 경우

예시)

```text
f(n) = f(n-1) + f(n-2)
```

# 💾 메모이제이션 (Memoization)

메모이제이션은 **재귀 과정에서 계산한 결과를 저장하여 재사용하는 방법**이다.

동일한 계산이 발생하면 저장된 값을 바로 사용한다.

### 특징

* Top-Down 방식
* 재귀 사용
* 계산 결과 저장
* 중복 계산 제거

👉 핵심: **"필요한 순간 계산하고 저장한다."**

---

## 📌 메모이제이션 흐름

1. 현재 값 계산 요청
2. 저장된 값 확인
3. 있으면 반환
4. 없으면 계산
5. 결과 저장 후 반환

---

# ⬇️ Top-Down

Top-Down은 큰 문제부터 시작하여 작은 문제로 내려가는 방식이다.

주로 재귀와 메모이제이션을 함께 사용한다.

### 예시

```java
int fib(int n) {
    if (n <= 1) return n;

    if (dp[n] != 0)
        return dp[n];

    return dp[n] = fib(n - 1) + fib(n - 2);
}
```

👉 핵심: **"필요한 값만 계산한다."**

---

# ⬆️ Bottom-Up

Bottom-Up은 가장 작은 문제부터 차례대로 해결하는 방식이다.

주로 반복문을 사용한다.

### 예시

```java
dp[0] = 0;
dp[1] = 1;

for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

👉 핵심: **"작은 문제부터 차근차근 계산한다."**

---

# 🔄 Top-Down vs Bottom-Up

| 구분    | Top-Down  | Bottom-Up |
| ----- | --------- | --------- |
| 구현    | 재귀        | 반복문       |
| 계산 방식 | 필요한 값만 계산 | 순서대로 계산   |
| 메모리   | 호출 스택 사용  | 호출 스택 없음  |
| 속도    | 상대적으로 느림  | 상대적으로 빠름  |
| 대표 기법 | 메모이제이션    | 테이블 채우기   |

# 🧩 점화식 (Recurrence Relation)

DP 문제의 핵심은 **점화식을 세우는 것**이다.

현재 상태를 이전 상태로 표현해야 한다.

### 예시

피보나치 수열

```text
f(n) = f(n-1) + f(n-2)
```

현재 값은 이전 두 값의 합으로 표현된다.

👉 핵심: **"현재 상태를 이전 상태로 표현한다."**

---

# 🗂️ DP 배열

DP에서는 계산 결과를 저장하기 위해 배열 또는 리스트를 사용한다.

예시)

```java
int[] dp = new int[n + 1];
```

* dp[i] : i번째 상태의 최적해
* 이미 계산한 값을 저장
* 중복 계산 제거

👉 핵심: **"계산 결과를 저장하는 공간"**

---

| 메모이제이션               | 중복 계산 제거          |
| Top-Down / Bottom-Up | 구현 방식 선택          |
| 시간복잡도                | 계산 횟수 분석          |
