# 🔁 재귀 (Recursion)

재귀는 **함수가 자기 자신을 호출하여 문제를 해결하는 방식**이다.

큰 문제를 작은 문제로 나누어 해결할 수 있으며, DFS 구현의 대표적인 방법으로 사용된다.

- 함수가 자기 자신을 호출
- 종료 조건(Base Case) 필수
- 현재 상태를 매개변수로 전달
- 호출 스택을 이용하여 동작

👉 핵심: **"문제를 더 작은 문제로 나누어 해결한다."**

---

## 🛠️ 재귀 핵심 요소

| 개념 | 설명 |
|------|------|
| 종료 조건 | 재귀를 멈추는 조건 |
| 자기 호출 | 함수 내부에서 자기 자신 호출 |
| 상태 전달 | 현재 탐색 상태를 인자로 전달 |
| 호출 스택 | 이전 상태를 자동으로 저장 |

---

## 📌 재귀 흐름

1. 현재 상태 처리
2. 더 작은 문제로 재귀 호출
3. 종료 조건 도달
4. 호출 순서의 역순으로 복귀

---

# 🌲 DFS (Depth First Search)

DFS는 **한 방향으로 가능한 깊게 탐색한 후 되돌아오는 탐색 방식**이다.

주로 재귀를 이용하여 구현하며, 완전 탐색과 경로 탐색 문제에서 자주 사용된다.

- Depth First Search
- 깊이 우선 탐색
- 재귀 또는 Stack 사용
- 모든 경우의 수 탐색 가능

👉 핵심: **"일단 끝까지 가본다."**

---

## 🛠️ DFS 핵심 요소

| 개념 | 설명 |
|------|------|
| 방문 처리 | 중복 탐색 방지 |
| 재귀 호출 | 다음 노드 탐색 |
| 종료 조건 | 탐색 종료 |
| 상태 관리 | 현재 탐색 정보 저장 |

---

## 💡 DFS 사용 예시

- 그래프 탐색
- 경로 탐색
- 완전 탐색
- 조합 / 순열
- 백트래킹

---

# 🔙 백트래킹 (Backtracking)

백트래킹은 **모든 경우를 탐색하되, 불필요한 경우는 더 이상 탐색하지 않고 되돌아가는 기법**이다.

DFS와 재귀를 기반으로 구현된다.

👉 핵심: **"가능한 경우를 탐색하고, 아니면 되돌아간다."**

---

## 🛠️ 백트래킹 핵심 요소

| 개념 | 설명 |
|------|------|
| 선택 | 현재 경우 선택 |
| 재귀 호출 | 다음 경우 탐색 |
| 복구 | 이전 상태로 되돌리기 |
| 가지치기 | 불필요한 탐색 제거 |

---

## 📌 백트래킹 흐름

1. 현재 선택 수행
2. 다음 상태 탐색
3. 탐색 종료
4. 원래 상태로 복구
5. 다른 선택 탐색

---

# ✂️ 가지치기 (Pruning)

가지치기는 **정답이 될 가능성이 없는 경우를 미리 제거하여 탐색량을 줄이는 기법**이다.

백트래킹 문제에서 매우 중요하다.

### 예시

- N-Queen에서 같은 열에 퀸이 존재
- 현재 늑대 수가 양 수 이상
- 남은 자원으로 목표 달성 불가능

👉 핵심: **"정답이 될 수 없는 경로는 탐색하지 않는다."**

---

# 🔄 DFS vs 백트래킹

| 구분 | DFS | 백트래킹 |
|--------|--------|--------|
| 목적 | 탐색 | 모든 경우 탐색 |
| 구현 | 재귀 / Stack | 재귀 |
| 상태 복구 | 필요 없음 | 필요 |
| 가지치기 | 일반적으로 없음 | 적극 활용 |
| 대표 문제 | 그래프 탐색 | N-Queen, 순열, 조합 |

---

# ⚠️ DFS / 재귀 문제에서 중요하게 볼 것

| 체크 포인트 | 설명 |
|-------------|------|
| 종료 조건 | 무한 재귀 방지 |
| 방문 처리 | 중복 탐색 방지 |
| 상태 관리 | 현재 탐색 상태 저장 |
| 상태 복구 | 백트래킹 여부 확인 |
| 가지치기 | 탐색 범위 최적화 |
| 시간복잡도 | 경우의 수 계산 |

---

# 💡 정리

| 개념 | 핵심 특징 |
|--------|--------|
| 재귀 | 자기 자신을 호출 |
| DFS | 깊게 탐색 |
| 백트래킹 | 탐색 후 상태 복구 |
| 가지치기 | 불필요한 탐색 제거 |
| 방문 배열 | 중복 탐색 방지 |
| 상태 관리 | 현재 탐색 정보 저장 |

### 이번 주 핵심

> DFS는 "탐색"에 집중하고,  
> 백트래킹은 "모든 경우를 효율적으로 탐색"하는 데 집중한다.

재귀 호출의 흐름을 이해하고, 상태를 관리하며 복구하는 과정을 익히는 것이 이번 주 목표이다.
