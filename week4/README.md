# 📚 스택(Stack)

스택은 **나중에 들어온 데이터가 먼저 나가는(LIFO)** 자료구조이다.

- Last In First Out
- 가장 위(top)의 데이터만 접근 가능
- 되돌리기(Undo), 괄호 검사 등에 자주 사용

👉 핵심: **"최근에 넣은 데이터를 가장 먼저 처리한다."**

---

## 🛠️ 주요 연산

| 기능 | 메서드 |
|------|--------|
| 삽입 | push() |
| 제거 | pop() |
| 조회 | peek() |
| 비었는지 확인 | isEmpty() |

---

## 💡 스택 사용 예시

- 괄호 검사
- 뒤로가기 / 실행 취소
- DFS
- 단조 스택(Monotonic Stack)

---

# 📦 큐(Queue)

큐는 **먼저 들어온 데이터가 먼저 나가는(FIFO)** 자료구조이다.

- First In First Out
- 한쪽에서 삽입, 반대쪽에서 제거
- 순서대로 처리해야 하는 상황에서 사용

👉 핵심: **"먼저 들어온 데이터를 먼저 처리한다."**

---

## 🛠️ 주요 연산

| 기능 | 메서드 |
|------|--------|
| 삽입 | offer() |
| 제거 | poll() |
| 조회 | peek() |
| 비었는지 확인 | isEmpty() |

---

## 💡 큐 사용 예시

- BFS
- 작업 대기열
- 순차 처리 문제
- 시뮬레이션 문제

---

# 🔄 Queue vs Stack

| 자료구조 | 특징 |
|----------|------|
| Stack | 후입선출 (LIFO) |
| Queue | 선입선출 (FIFO) |

---

# ⚠️ 구현체 고민하기

자바에서는 인터페이스와 구현체 차이를 이해하는 것이 중요하다.

| 인터페이스 | 구현체 |
|------------|--------|
| Stack | Stack |
| Queue | LinkedList, ArrayDeque |
| Deque | ArrayDeque, LinkedList |

👉 핵심:
- Queue는 인터페이스라 바로 생성 불가능
- ArrayDeque가 대부분의 Queue/Stack 상황에서 빠름
- LinkedList는 노드 기반 구조

---

# 💡 정리

| 자료구조 | 핵심 특징 |
|----------|-----------|
| Stack | 최근 데이터 우선 처리 |
| Queue | 먼저 들어온 데이터 우선 처리 |
| Deque | 양쪽 삽입/삭제 가능 |
| 구현체 선택 | 상황별 시간복잡도와 메모리 고려 |
