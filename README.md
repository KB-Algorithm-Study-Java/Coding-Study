## 📁 프로젝트 구조

본 스터디의 저장소는 `algorithm-study`를 루트로 하며, 아래와 같은 구조로 구성됩니다.

```
algorithm-study/
├── week0/
│   ├── README.md
│   ├── 문제명 or 문제 번호/
│   │   ├── KJW/
│   │   │   └── Main.java
│   │   └── ...
│   ├── 문제명 or 문제 번호/
│   ├── 문제명 or 문제 번호/
│   └── challenge/
│
├── week1/
│   ├── README.md
│   ├── 문제명 or 문제 번호/
│   ├── 문제명 or 문제 번호/
│   ├── 문제명 or 문제 번호/
│   └── challenge/
```

---

## 📚 주차별 구성

* 각 `weekN` 폴더는 해당 주차의 학습 내용을 담습니다.
* 각 주차 폴더에는 다음이 포함됩니다:

    * `README.md`
      → 해당 주차에 학습한 알고리즘 개념 및 회의 내용 정리
    * **필수 문제 3개**
    * **도전 문제 1개 (challenge)**

---

## 👨‍💻 문제 풀이 방식

* 각 문제 폴더 내부에는 **팀원 이름 폴더**를 생성합니다.

```
문제폴더/
├── KJW/
│   └── Main.java
├── KHS/
└── PSH/
```

* 문제를 풀이한 후:

    * 반드시 `Main.java` 파일명으로 코드 작성
    * 본인 폴더에만 업로드

---

## 📝 추가 기록 (중요)

* 각 문제 폴더 내에 `README.md` 파일을 추가하여 아래 내용을 작성합니다:

```
- 문제 접근 방식
- 문제 해결 아이디어

- 맞은 화면 캡쳐 인증
```

## 🌿 Git 협업 규칙

### 🔹 브랜치 전략

* `main` 브랜치에 직접 push ❌
* 본인 이름 **초성 브랜치 생성 후 작업**

```
예: KJW, PSH, KHS
```

---

### 🔹 작업 흐름

1. 본인 브랜치 생성
2. 문제 풀이
3. commit
4. 본인 브랜치에 push

---

```

```

