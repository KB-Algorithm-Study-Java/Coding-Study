# 📌 1주차 - 구현 / 완전탐색
### * 이번 주는 각자 선택한 세 문제 Solve
## 🛠️ 구현 (Implementation)

구현은 문제에서 요구하는 조건을 그대로 코드로 옮기는 유형이다.
알고리즘 자체의 난이도보다는 **문제를 정확히 이해하고 실수 없이 코드로 표현하는 능력**이 중요하다.

* 입력 조건, 출력 형식, 예외 처리 등을 꼼꼼히 확인해야 한다.
* 사소한 조건 하나 때문에 오답이 나는 경우가 많다.
* 문자열 처리, 시뮬레이션, 배열/리스트 조작 등이 자주 등장한다.

👉 핵심: **“문제를 있는 그대로 정확하게 구현하기”**

---

## 🔍 완전탐색 (Brute Force)

완전탐색은 가능한 모든 경우를 전부 확인하여 정답을 찾는 방법이다.
경우의 수가 많아 보이더라도, **문제의 제한을 보고 충분히 가능한지 판단하는 것이 중요하다.**

* 모든 경우를 탐색하므로 정답을 놓칠 가능성이 없다.
* 반복문, 재귀, 백트래킹 등을 활용한다.
* 시간복잡도를 반드시 고려해야 한다.

👉 핵심: **“가능한 모든 경우를 확인하되, 시간 안에 가능한지 판단하기”**
